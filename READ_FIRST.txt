STEP 2: REGISTER - Signup करो (5 minutes)

Option A: Full Signup (recommended)

A. GitHub signup (2 minutes):
   1. Go to: https://github.com/signup
   2. Enter: Username, Email, Password
   3. Verify email (check inbox)

B. Create Repository on GitHub (2 minutes):

STEP 3: UPLOAD - Push code to GitHub (10 minutes)
──────────────────────────────────────────────────────────────────────────────────────

Open Command Prompt (cmd.exe or PowerShell)

Copy these commands (one by one):

   cd c:\Users\user\Downloads\veerbot
   
   git config --global user.name "Your Name"
   git config --global user.email "your-email@gmail.com"
   
   git init
   git add .
   git commit -m "VEERBOT Cloud Deployment"
   git branch -M main
   git remote add origin https://github.com/YOUR-USERNAME/veerbot.git
   git push -u origin main

⚠️ IMPORTANT: Replace "YOUR-USERNAME" with your actual GitHub username
   For example: if your username is "raj123"
   Change to: https://github.com/raj123/veerbot.git


STEP 4: DEPLOY - Deploy on Render (auto, 5 minutes)
──────────────────────────────────────────────────────────────────────────────────────

1. Go to: https://render.com/dashboard
2. Click "New +" → "Web Service"
3. Select your veerbot repository
4. Fill these:
   Name: veerbot
   Runtime: Python 3.11
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn app:app
5. Click "Create Web Service"
6. Wait 3-5 minutes
7. ✅ DONE! You'll get a URL like: https://veerbot-xyz123.onrender.com

❓ COMMON ISSUES & FIXES:
═══════════════════════════════════════════════════════════════════════════════════════

Issue: "git command not found"
Fix: Install Git from https://git-scm.com/download/win

Issue: Deploy failed on Render
Fix: Check Render logs (dashboard → logs) - shows error message

Issue: Bot URL doesn't work
Fix: Wait 5 min or refresh → Render still deploying

Issue: Bot responds slowly
Fix: Normal on free tier! First request slower, then fast

Issue: GitHub username/password wrong
Fix: Use personal token from https://github.com/settings/tokens

Issue: Repository already exists
Fix: Use different name or delete old repo
